# 1. 概要

本ソフトは、小説投稿・公募応募前のレイアウト確認および文章チェックを目的とした、縦書き小説ビューアである。

- 単一HTMLファイルで動作する自己完結型アプリケーション
- テキストファイル（.txt）を開いて閲覧・確認する
- 公募原稿の組版確認・校正支援を主目的とする
- PC・スマートフォン・タブレット対応（レスポンシブ）

対応文字数

- 必須：20万文字
- ベストエフォート：100万文字

# 3. レイアウト

## 3.1 ページ形式

ページ形式はプリセットから選択できる。

### 初期搭載プリセット

| 名称 | 文字数 |
|------|-------:|
| 20字 × 20行 | 400字 |
| 40字 × 16行 | 640字 |
| 40字 × 32行 | 1280字 |
| 40字 × 34行 | 1360字 |
| 42字 × 34行 | 1428字 |
| カスタム | 任意 |

プリセット名には1ページあたりの文字数も表示する。

例

```
40字 × 16行（640字）
```

---

## 3.2 カスタム設定

任意に

- 1列の文字数
- 1ページの列数

を指定できる。

---

## 3.3 レイアウト方針

ページ内容は文字数で決定する。

ウィンドウサイズ変更では

- ページ内容は変えない
- フォントサイズ
- 行間
- セルサイズ

のみ自動調整する。

ページ境界が変わるのは以下の場合のみ。

- ページ形式変更
- 禁則設定変更
- ぶら下げ変更

ルビ表示ON/OFFではページ境界を変えない（再描画とワーニング再検出のみ。ルビはマス数に影響しないため。5.3参照）。

---

## 3.4 設定保存

以下を LocalStorage に保存する。

- ページ形式
- テーマ
- フォント種別
- フォントサイズ
- 禁則設定
- ぶら下げ設定
- ルビ表示
- 半角文字色表示

次回起動時に復元する。

# 4. ページ分割

## 4.1 文字カウント

| 対象 | カウント |
|------|---------|
| 全角文字 | 1マス |
| 全角スペース | 1マス |
| 半角文字 | 0.5マス（4.2参照） |
| 改行 | 強制改列 |
| ルビ | カウントしない |

---

## 4.2 半角文字

英数字・半角記号・半角カナは

- 常に横倒し表示
- 縦中横は行わない

**占有幅は全角の1/2（0.5マス）とする。**

- 半角文字は1文字あたり0.5マスを占有する。
- 半角文字列は1つのトークンとして扱い、トークンの占有マス数は「半角文字数 × 0.5」を**切り上げ**て整数マスに丸める（列内マス割りの整合のため）。
- この規則により、ページ分割はWorker内で文字列のみから決定的に計算できる（DOM計測不要）。

# 5. 組版オプション

すべて ON / OFF 切替可能。

---

## 5.1 禁則処理

### ON

- 行頭禁則
- 行末禁則

を適用する。

### OFF

指定文字数で機械的に分割する。半角トークンが列末をまたぐ場合は、トークンを分割せず次列へ送る。

禁則文字一覧は編集可能。

---

## 5.2 ぶら下げ

禁則ON時のみ有効。

### ON

句読点をぶら下げ表示。

### OFF

通常禁則。

---

## 5.3 ルビ

対応形式（カクヨム）

```text
｜漢字《ルビ》
漢字《ルビ》
```

### 配置・カウント規則

- ルビは親文字の脇に表示する。
- **ルビは親文字より長くても字送り（マス数）にはカウントしない。** ルビが親文字幅をはみ出す場合もページ分割・列マス割りには影響させない（表示上のはみ出しは許容する）。
- ルビ本体は親文字トークンに付随して同一列内に配置し、列またぎはさせない。

### ON

ルビを親文字の脇に表示する。

### OFF

ルビを振らず、**ルビ記法（`｜` `《` `》`）も含めた元テキストをそのまま本文として表示する**（例：`｜漢字《かんじ》` は `｜漢字《かんじ》` と地の文の並びで表示）。あわせてワーニング一覧へ登録する。

ルビ表示のON/OFFはページ境界に影響しない（マス数不変）。切替時は再描画とワーニング再検出のみ行い、再ページ分割はしない。

---

## 5.4 半角文字

常に横倒し表示。

### ON

色付き表示。

### OFF

通常色。

ワーニング対象には常に含める。

---

# 6. 見出し

Markdown形式を採用する。

```text
# 第一章
## 第一話
```

認識内容

- `#`：章
- `##`：話

### 表示方針

- 見出し行は**色付き・強調表示（太字）**のみを行う。
- **組版・レイアウトは通常本文と同一とし、元テキストのフォーマットを維持する。** 見出し専用の改列・専有行・字送り変更は行わない。
- `#` 記号も本文にそのまま表示する。
- 文字カウントも通常本文と同じ規則で行う（見出しだからといって特別扱いしない）。

サムネイル・スクロールバーにも色付きマーカーを表示する。

# 12. 原稿ファイル自動更新

File System Access API 対応ブラウザ（Chrome/Edge等）では、保持したファイルハンドルを一定間隔または再フォーカス時に読み直し、更新を確認して再読み込みする。現在位置は保持する。
非対応ブラウザ（Safari/Firefox等）および `file://` 実行時は手動更新ボタン（再選択）を提供する。

> 「自動リロード」は実機での挙動差が大きいため、実装・検証が済むまでは「対応環境での更新確認」として控えめに扱う。`<input type="file">` によるフォールバックは常に残す。

# 17. 確定仕様（実装反映・v6 / FIX）

本章は実装済みの最終仕様を確定するもの。以前の章と差異がある箇所は本章を優先する。
プロダクト名：**縦view**、配布ファイル：**TateView.html**（単一HTML・依存ゼロ・ビルド不要で利用）。

## 17.1 画面構成

- **上部メニュー（1段）**：ロゴ「縦view」／「開く」／ファイル情報（ファイル名・文字コード・改行コード・総文字数）／検索窓・🔍・前・次・件数／「更新」ボタン（自動更新可能時は非ボタンの「自動更新」表示）／体裁プリセット／字（1列字数）／行（1ページ列数）／「校正」ボタン（強調表示）。
- **左サイドバー**：上にサムネイル一覧、下に操作メニュー（ページ前/次、ページ番号ジャンプ＋総ページ数、現ページ文字数／総文字数、進捗メッセージ、禁則・ぶら下げ・ルビ・半角色・升目、テーマ、書体・字級）。
- **本文領域**：中央に縦書き表示。
- **校正パネル**：右側にトグル表示（severity 色分け、一覧クリックで該当箇所へジャンプ、前後移動）。
- 下部バーは持たない。

## 17.2 入力

- 文字コードは自動判定のみ（UTF-8 BOM有/無・Shift_JIS）。手動選択UIは持たない。判定不能時はファイル情報に「文字コード判定不可」と表示。
- **ドラッグ＆ドロップ**でファイルを開ける（対応ブラウザではファイルハンドルを取得し自動更新監視も有効化）。

## 17.3 レイアウト・表示

- ページ内容は文字数で決定（DOM計測非依存）。**空行は通常幅の空白列（原稿用紙の空行）として表示**する。
- ウィンドウに対し、指定した字数×行数が必ず収まるよう、はみ出す場合はページ全体を等倍縮小して全列を表示する。
- 字級は「表示可能なサイズ」のみ選択可能（収まらないサイズは提示しない）。
- 半角は常に横倒し（`text-orientation: sideways`）。縦中横は行わない。
- 見出し `#`（章）と `##`（話）は色を分けて強調。組版は本文と同一・記号もそのまま表示。
- **テキスト選択・コピーは親文字のみ**に正規化（ルビ・ルビ記法を含めない）。
- ルビOFF時はルビ記法（`｜《》`）を含む元テキストをそのまま表示。

## 17.4 升目（原稿用紙モード）

- ON時、各列を「1列字数 × セル間隔」の全長に固定し、**文字の無いマスにも縦横の罫線**を引く。
- 目盛り：ページ**右側（升目の外）に文字位置**、**上側に列位置**を、それぞれ 5 単位で一列だけ表示（右は半文字上、上は半文字左にオフセット）。

## 17.5 ナビゲーション

- ページ移動は左サイドバーの「◀ 次」「前 ▶」（縦書きは左進行）、ページ番号ジャンプ、キー（←＝次）、ホイール、スワイプ。
- サムネイルはクリック／ドラッグでページ切替。見出しは小さくラベル表示。
- 現在位置は文字オフセット＋周辺文字列で保持（ファイル名基準キー）。

## 17.6 原稿ファイル更新

- File System Access 対応環境：ファイル変更を検知して自動再読込（現在位置保持）。上部に「自動更新」表示。再読込時は「更新された」を約1秒トースト表示。
- 非対応・`file://`：「更新」ボタンで手動再読込。

## 17.7 AIチェック

- 今回スコープ外（UI・APIキー入力欄とも実装しない）。

## 17.8 技術・品質

- 単一HTML。重い処理は Web Worker、`documentId`/`requestId` で世代管理（古い結果は破棄）。
- エンジン（判定/正規化/トークナイズ/分割/警告/検索/位置）は純粋関数として `node --test` で回帰テスト（全緑）。ページ分割は半開区間・不変条件つき。
- ビルド：`node build.js`（main/worker 2バンドルを単一HTML化、外部参照なしを検証）。
