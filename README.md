# 縦View（TateView）— 縦書き小説ビューア

配布ファイル名は `TateView.html`。

公募・投稿前の小説原稿を、縦書きで組版確認・校正するためのビューア。
テキストファイル（.txt）を開くだけで、原稿用紙のような縦書きレイアウトと、公募向けの文章チェックが行える。**単一HTMLファイルで動作し、インストール不要・オフライン利用可**。

---

## 特徴

- **縦書き表示**：文字数ベースのページ分割で、ウィンドウサイズを変えてもページ内容（ページ割り）が動かない。収まらない場合はページ全体を等倍縮小し、指定した行数を必ず表示する。
- **公募向けプリセット**：20字×20行（400字）〜42字×20行（840字）など。カスタム指定も可。
- **原稿用紙（升目）モード**：空マスにも縦横の罫線を引き、ページ外周に5単位の目盛り（右＝文字位置／上＝列位置）を表示。
- **組版オプション**：禁則処理・ぶら下げ・ルビ表示・半角文字の色表示を個別にON/OFF。空行は空白列として表示。
- **校正ワーニング**：字下げ漏れ、約物の使い方、三点リーダ／ダッシュの偶数チェックなど15項目を検出。重要度（error/warning/info）で色分けし、一覧クリックで該当箇所へジャンプ。
- **全文検索**：件数表示・前後移動・強調表示。
- **サムネイル**：左側にページ一覧（見出しを小さくラベル表示）。クリックまたはドラッグでページ切替。
- **位置復帰**：ページ番号でなく文字位置で保持。原稿を編集しても近い位置に戻る。
- **原稿の更新**：File System Access 対応ブラウザ（Chrome/Edge）ではファイル更新を自動検知して再読み込みし「更新された」を表示（現在位置は保持）。非対応環境は「更新」ボタンで手動。
- **ドラッグ＆ドロップ**でファイルを開ける。
- **テーマ／フォント**：ライト・ダーク、明朝・ゴシック切替、字級（表示可能なサイズのみ選択可）。テキスト選択・コピー対応（コピーは親文字のみ）。
- 見出し `#`（章）・`##`（話）を色分け強調。
- 文字コード（UTF-8 / Shift_JIS）と改行コード（CRLF / LF / CR）を自動判定して表示。

対応文字数：20万文字で快適動作、100万文字までベストエフォート。

---

## 使い方

1. `dist/TateView.html` をブラウザで開く（ダブルクリックでOK）。
2. 「開く」から `.txt` 原稿を選択、または画面にドラッグ＆ドロップ。
3. 上部メニューで体裁（字数×行数）・検索・校正、左メニューでページ移動・組版・テーマ・書体・字級を操作。
4. ページ送りはキー（←＝次）／スペース／ホイール／スワイプ、左のサムネイル、ページ番号入力から。校正パネルや検索から該当箇所へジャンプ。

対応ブラウザ：Chrome / Edge / Firefox / Safari（PC）、iPhone / Android、タブレット。
※ ファイルの自動更新は File System Access API 対応ブラウザ（Chrome/Edge）のみ。非対応環境や `file://` 実行時は「更新」ボタン（再選択）を使う。`file://` で Worker が起動しない場合はローカルサーバ経由で開く。

## ルビ記法

カクヨム互換の記法に対応。

```
｜漢字《かんじ》     ← ｜で親文字の範囲を明示
漢字《かんじ》        ← 直前の漢字列を自動で親文字に
```

ルビは字送り（マス数）に影響しない。親文字より長くても表示上はみ出すが、ページ境界は変わらない。ルビ表示をOFFにすると、ルビ記法（`｜《》`）を含む元テキストがそのまま本文として表示される。

## 見出し記法

Markdown 風。組版は本文のままで、色と太字で強調される（`#` 記号もそのまま表示）。

```
# 第一章
## 第一話
```

## 開発者向け

### 構成

```
src/        開発用ソース（ES Modules・複数ファイル）
  modules/  副作用のない純粋関数（normalize, encoding, kinsoku, tokenizer, paginator, warnings, search, position）
  ui/       renderer（描画）, settings（保存）
  worker.js 重い処理を担う Web Worker
  main.js   メインスレッド（UI・ナビ・サムネイル・自動更新を集約）
tests/      node --test で動くテスト（engine.test.mjs・32件）
sample/     サンプル原稿（UTF-8 / Shift_JIS）
build.js    src を単一HTMLに結合（main/worker 2バンドル）
dist/       ビルド成果物 TateView.html
```

### 設計方針

重い処理（文字コード判定・正規化・トークナイズ・ページ分割・ワーニング・検索）はすべて Web Worker に隔離し、メインスレッドは表示ページのみを描画する。ページ分割はDOM計測に一切依存せず、「全角=1マス／半角=0.5マス（切り上げ）」の文字数モデルだけで決定的に計算する。これによりウィンドウ幅や端末が変わっても同じ原稿は必ず同じページ割りになる。

内部オフセットは正規化済みテキスト上の UTF-16 コードユニット位置に統一し、範囲はすべて半開区間 `[start, end)` で扱う。トークナイザはストリーミング（generator）で全Token配列を保持しない。Worker通信は documentId/requestId による世代管理で古い結果を破棄する。

外部ライブラリは使わない（依存ゼロ）。詳細な実装ルール・アルゴリズム・型契約・受け入れ基準は `AGENTS.md` を参照。

### ビルド

```bash
node build.js        # src/ を dist/TateView.html に結合（Node標準モジュールのみ）
```

### テスト

```bash
node --test          # エンジンの回帰テスト（32件）
```

ページ分割は最重要機能のため、既知原稿に対する期待ページ境界を固定した回帰テストと、同一入力での決定性テストを必ず維持すること。
